# 🌿 EcoScore — Carbon Awareness Chrome Extension

> **PromptWars Submission** · Carbon *awareness*, not carbon *tracking*.

EcoScore intercepts you **before** you confirm a food order, cab booking, or flight — shows you the real carbon cost in terms you feel (not just kg numbers), and ties your choices to a credit score that your whole city can see.

---

## The Problem This Solves

Carbon footprint tools have a UX problem: they show you data *after* decisions are made, in abstract units (kg CO₂e) that mean nothing emotionally. Nobody changes behaviour because a dashboard turns red.

**EcoScore intervenes at the moment of decision** — on Zomato checkout, before you confirm an Ola ride, when you're about to book a flight — with:

1. **An emotional analogy** generated by Gemini: *"This chicken burger = leaving your AC on for 3 days straight."*
2. **A concrete alternative**: *"Dal Makhani instead — saves 1.4 kg CO₂."*
3. **A credit score impact**: *"−12 EcoCredits if you proceed. +8 if you switch."*

The leaderboard creates social accountability — your rank in Bangalore this month is visible, resettable, and shareable.

## Setup

### 1. Get a free Gemini API key

1. Go to [Google AI Studio](https://aistudio.google.com/app/apikey)
2. Create a new API key (free tier: 1,500 requests/day)

### 2. Load the extension

1. Open Chrome → `chrome://extensions`
2. Enable **Developer mode** (top right toggle)
3. Click **Load unpacked**
4. Select this `ecoscore/` folder

### 3. Add your API key

1. Click the EcoScore icon in Chrome toolbar
2. You'll be prompted to open settings
3. Paste your Gemini API key → **Save key** → **Test connection**

### 4. Start browsing

Visit any supported site:
- **Food**: zomato.com, swiggy.com
- **Cabs**: olacabs.com, uber.com, rapido.bike, blusmart.com
- **Shopping**: amazon.in, flipkart.com, myntra.com
- **Travel**: makemytrip.com, ixigo.com, goibibo.com

The banner appears automatically before checkout.

## How Credits Work

| Action | Credits | Why |
|---|---|---|
| Metro / bus ticket detected | +25 | 0.03 kg CO₂ vs 2.1 kg cab |
| Shared cab (Ola Share/Pool) | +15 | Splits emissions |
| EV cab (BluSmart) | +20 | Zero tailpipe |
| Plant-based meal ordered | +18 | 50% lower than chicken |
| Accepted green alternative | +10 | Switched on Gemini nudge |
| Solo cab — short trip (<3km) | −12 | Walkable, high per-km cost |
| Beef/lamb ordered | −20 | 27 kg CO₂e per kg — highest |
| Express delivery chosen | −15 | Air freight = 50× sea freight |
| Domestic flight booked | −60 | Train = 10× less CO₂ |
| Ignored nudge (proceeded) | 0 | Awareness, not punishment |

Score starts at **500** (like a credit score). Monthly leaderboard resets on the 1st.

## Grading Criteria Evidence

### Code Quality
- Manifest V3 with strict service worker pattern
- Modular architecture: each concern in its own file (gemini.js, score.js, banner.js, per-site content scripts)
- JSDoc comments throughout
- ESM modules with named exports
- No global state pollution — debounce timers namespaced under `window.__ecoscore_*`
- All selectors use fallback arrays for resilience against site DOM updates

### Security
- **API key never hardcoded** — stored in `chrome.storage.local`, entered by user via options page
- **Content Security Policy** enforced in manifest: `script-src 'self'; object-src 'none'`
- **Shadow DOM** for banner injection — style isolation prevents XSS via host page CSS
- All string injection uses `textContent` (not `innerHTML`) — zero XSS surface in banner
- `sanitizeForPrompt()` strips control chars, HTML chars, and limits length before Gemini prompt
- Google OAuth token fetched from Identity API, not stored manually

### Efficiency
- **Debounced analysis**: 800ms–1200ms debounce on MutationObserver prevents excessive calls
- **60-second cooldown** per item in background service worker via `pendingAnalysis` Map
- **24-hour local cache** for Gemini results — repeat orders skip the API entirely
- Service worker stays dormant when idle (MV3 default)
- Badge updates batched with score writes

### Testing
- **230 unit tests** across 8 test suites (covering both root and `ecoscore/` folder)
- `gemini.test.js`: sanitization, JSON parsing, cache keys, field validation, delta clamping
- `score.test.js`: tier boundaries, delta messages, streak logic, monthly summaries, history trimming
- `dom-mock.test.js`: food/cab/travel/e-commerce detection logic against simulated DOM structures
- `security.test.js`: dedicated security tests verifying input sanitization, origin/SSRF checking, schema validation, and cache key safety
- All tests run with `npm test`

### Problem Statement Alignment
- **No dashboard** — the popup shows a score and 8 recent actions, nothing more
- **Decision intercept** — banner fires before checkout, not after
- **Emotional framing** — Gemini generates analogies in Indian everyday-life terms
- **Credit score psychology** — mirrors CIBIL score, starts at 500, people are trained to care
- **Leaderboard mechanic** — monthly reset, city-scoped rank, shareable rank card
- **Zero penalty for ignoring** — awareness without punishment keeps engagement high
- **Invisible sources covered**: food delivery packaging waste, air freight express delivery, solo cab deadheading, short-haul flights vs train

## Invisible Carbon Sources Researched

Beyond the obvious, EcoScore targets these often-ignored emission sources:

1. **Food delivery packaging** — plastic containers, bags, cutlery per order
2. **Last-mile delivery vans** — cold-chain vehicles for dairy/meat orders
3. **Express e-commerce air freight** — same-day/2hr delivery travels by air
4. **Cab deadheading** — empty return trips after drop-off (included in per-km estimates)
5. **Short-haul flights** — BLR↔HYD or BLR↔CHN are vastly better by train
6. **Festival buying surges** — aggregated demand spikes requiring extra inventory logistics
7. **Fast fashion** — synthetic fabrics (polyester, nylon) have 2× the footprint of cotton
8. **Electronics purchases** — manufacturing + shipping dominates lifecycle emissions
